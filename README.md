# Product-Decision
SQL to answer a product question
the goal was to determine if our payment failure resolution should rely on a static rules engine or an ML-driven classifier. This analysis provided the quantitative evidence needed to pivot from a simple rules-based approach to an ML-first architecture by demonstrating:

Non-linear failure patterns that cannot be captured by simple thresholds.

Feature signal strength across different payment gateways.

Operational ROI by proving that an 81% Straight-Through Processing (STP) target was achievable.
